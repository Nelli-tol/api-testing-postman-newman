# API Test Strategy

## Objective
Validate correctness, stability, and error handling of REST API endpoints.

## Scope
- GET, POST requests
- Status code validation
- Response body validation
- Negative scenarios (invalid input, missing data)

## Out of Scope
- Performance testing
- Security testing
- Load testing

## Test Types
- Functional testing
- Negative testing
- Smoke API checks

## Tools
- Postman for request execution
- Newman for command-line execution

## Environment
- Public demo REST APIs
